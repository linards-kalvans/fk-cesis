Weeks 9–14 · 35h

Testing checklist
Create a test training event in Dolibarr — n8n detects it within the hour
Poll message arrives in test WhatsApp group at correct time with correct event details
Reply "1" (coming) from a test number — attendance record created in Dolibarr within 30 seconds
Reply "2" (not coming) — attendance record updated, not duplicated
Unknown number replies — workflow handles gracefully (logs warning, no crash)
Competition event: poll sent N days before (configurable), second poll 2 days before
Month-end coach billing: create 10 test attendance records → billing workflow calculates correctly → draft expense in InvoiceNinja matches expected amount

Run live with one squad for 2 full weeks before expanding to all squads
Highest-risk phase. Meta verification timing is unpredictable — build in a 2-week buffer. WhatsApp message template approval (required for outbound messages) takes 1–5 business days per template. Test the full poll → parse → write-back loop with real phones before going live with parents. Response parsing edge cases (free-text replies, reactions instead of numbered responses) need explicit handling.

04 Risk register
Risk Severity Phase Mitigation
Meta business verification rejected or delayed beyond 2 weeks
HIGH
 Ph 3 Start verification in parallel with Phase 2. Have a fallback: send attendance polls via email or Telegram until WhatsApp is live. Don't block Phase 2 billing on it.
Stripe Latvia SEPA payments don't work as expected
HIGH
 Ph 1 Test with a real Latvian bank account in Phase 1 (test mode, then live with €1). If SEPA fails, fall back to card-only — most parents have a Visa/Mastercard. SEPA becomes optional.
WhatsApp response parsing breaks on free-text replies / emojis
MED
 Ph 3 Use numbered quick-reply buttons (WA interactive messages) instead of free-text parsing. Buttons are more reliable and require no NLP. Parse fallback: any unrecognised message is logged as "no response" and alerts admin.
DocTR OCR accuracy poor on Latvian ID cards
MED
 Ph 4 OCR prefill is a convenience, not a gate. All fields are editable before submission. Low accuracy = more manual review, not a system failure. If DocTR is consistently poor, swap to Eden AI API (paid, ~$0.01/doc) without changing the n8n workflow.
n8n workflow failure causes missed polls / unrecorded attendance
MED
 Ph 3–5 n8n has built-in error logging and retry logic. Add a Slack/email alert on any flow failure. For missed polls: coaches mark attendance manually in Dolibarr as fallback. Attendance is never fully automated — manual override always works.
Club treasurer can't operate system after handover
MED
 Ph 5 InvoiceNinja and Dolibarr are both operable without technical knowledge for day-to-day tasks. The n8n automation runs silently. Treasurer only needs to: check payment dashboard, add/remove members, send manual reminders. Write a 1-page cheat sheet for these 3 tasks.
skuby7 VPS runs out of RAM with all services running
LOW
 Ph 1 Stack requires ~2 GB peak RAM. Check current skuby7 spec — if under 4 GB, upgrade to Hetzner CX32 (~€8.50/mo). DocTR runs on-demand only so peak usage is brief.
LFF / COMET requires digital registration that conflicts with manual export
LOW
 Ph 4 Dolibarr has CSV/Excel export for member data. Map fields to LFF format once and document it. If LFF publishes an API later, n8n can handle the push. Not a blocker for any phase.
